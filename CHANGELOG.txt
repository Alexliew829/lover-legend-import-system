正式版 V2.0

同步机制重构：
- 所有产品、进口、库存、批次及汇率设置统一进入同步队列。
- 本地未同步状态会持久保存，断网、关闭页面或重新打开后仍会继续同步。
- 新增进口不会在首次拉取云端资料时被旧资料覆盖。
- 云端版本冲突时自动合并较新记录，再安全提交，不再强制覆盖。
- 连续保存产品、进口明细及批次只合并为一次同步。
- 同步进行中若再次修改，会保留第二次修改并只追加一次必要同步。
- 移除同步完成后的自动 Refresh，直接更新当前页面资料。
- Service Worker 改为更稳定的缓存策略，页面启动与重复打开更快。

保留：
- V1.9 原有页面排版。
- 原有进口成本、库存、日期及编号计算逻辑。

V2.5 Inventory Zero Fix
- 编辑现有进口编号时，当前剩余数量允许设为 0。
- 库存为 0 后，首页库存管理不再显示该产品，但进口历史继续保留。
- 当前剩余数量可在 0 至原进口数量之间增加或减少，用于销售、退货及盘点修正。
- 原进口数量、Average Cost、海外运费比例及原批次成本保持不变。
- Apps Script 无修改，不需要重新部署。

V2.5 Cumulative Inventory Limit Fix
- 编辑进口编号时，数量代表同一产品的当前总库存。
- 可编辑范围：0 至同一产品累计原进口数量。
- 例：第一批进口5棵，第二批再进口3棵，当前库存允许0至8，不允许9。
- 销售、退货、盘点只改变当前库存。
- 每张进口编号的原进口数量、原成本、Average Cost及海外运费比例保持不变。
- Code.gs 未修改，不需要重新部署 Apps Script。
V2.5 Batch Remaining Quantity Fix
- 编辑 A001 只显示并调整 A001 自己的剩余数量，不会显示同产品累计库存。
- A001 原进口 5、A002 原进口 3：编辑 A001 显示 5，编辑 A002 显示 3。
- 每个进口编号可编辑范围为 0 至该批次原进口数量。
- 产品总库存按该批次新旧剩余数量的差额自动加减。
- 原进口历史、Average Cost、海外运费比例及历史成本不变。
- Code.gs 未修改，不需要重新部署 Apps Script。

V2.5 Multiple Batch Display Fix
- 同一产品再次进口时始终生成新的独立进口编号。
- 首页同一产品显示所有仍有库存的进口编号及各批次当前剩余数量。
- 进口历史显示同产品其他进口编号、原进口、当前剩余与同产品总库存提示。
- 产品当前库存仍为所有批次剩余数量合计。
- Code.gs 未修改，不需要重新部署 Apps Script。
